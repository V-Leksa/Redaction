Приложение позволяет в зависимости от роли зарегистрированного пользователя использовать соответствующий функционал:
1) Администратор
данные администратора уже есть в системе, поэтому администратор может свободно авторизироваться. 
После авторизации выйдет сообщение: "Пользователи успешно загружены из файла JSON" - это значит, что данные пользователей загрузились в панель администратора из файла json, после нажатия на сообщении ОК, администратору откроется его панель. 
Верхняя половина.
Позволяет видеть всех зарегистированных пользователей в Списке пользователей. При выделении в Списке пользователей конкретного пользователя в окне Данные пользователя будут видны его личные данные при регистрации.
Кнопка Добавить нового пользователя откроет администратору панель регистрации, где он сможет добавить нового пользователя.
Кнопка Сохранить изменения. При выделенном пользователе в Списке пользователей в окне Данные пользователя можно корректировать и нажимать на кнопку Сохранить изменения. Сделана возможность менять пароль пользователя. Примечание разработчика: не совсем довольна тем как работает данная функция, поэтому просьба применять данную кнопку с осторожностью!
Кнопка Удалить пользователя. Также при выделении пользователя в Списке пользователей и нажатии данной кнопки удаляет порльзователя из Списка пользователей, кдаляет данные из окна Данные пользователя,также удаляет созданную при регистрации папку пользователя с его именем.
Нижняя половина.
Нажав на кнопку Отобразить список в окне Список статей и авторов отобразятся все папки авторов и их статьи в виде структуры Дерева. Нажав на статью в этом списке в окне Статья отобразится содержимое статьи.
Кнопка Удалить статью. При выделении статьи и нажатии кнопки будет удален файл статьи из папки пользователя, в окне Список статей и авторов данная статья исчезнет и в окне Статья содержимое статьи тоже удалится.
Кнопка Сохранить изменения. Также при выделении статьи, внесении измений в ее содержание с окне Статья и нажатии кнопки Сохранить изменения содержание файла со статьей перезапишется.
2)Автор
При регистрации необходимо отметить галочкой эту роль,чтобы при авторизации открылась панель Автора. Также при регистрации создастся папка с именем пользователя, куда будут сохраняться созданные статьи автора.
Панель позволяет создавать статьи, заполнив окна Название статьи и Текст статьи. После нажатия кнопки Создать статью файл со статьей будет сохранен в формате txt, а название статьи будет добавлено в Список статей.Также выйдет сообщение о создании статьи.
При выделении статьи в Списке статей в окне Текст статьи появится содержание статьи, если вносить изменения в статью, а потом нажать на кнопку Сохранить изменения, то содержимое файла со статьей перезапишется.
При выделении статьи в Списке статей и нажатии кнопки Удалить статью, файл со статьей будет удален из папки данного пользователя, а также название статьи удалится из Списка статей.
Чтобы снять выделение со статьи с окне Список статей нужно кликнуть мышью в любом месте панели.
3) Читатель
При регистрации и отметки о роли Читателя, пользователю открывается панель Читателя. При нажатии кнопки Отобразить список в окне Список авторов и статей появится список всех авторов и статей в виде Дерева. При выделении нужной статьи в окне Статья появится ее содержимое.

Данные по реализации:
При регистрации пользователя автоматически, вне зависимости от Роли создается папка для пользователя (на случай смены Роли - папка уже будет, при удалении пользователя папка также будет удалена). Имеется проверка пароля и почты на уникальность при регистрации.
Также при регистрации сохраняется путь к этой папке. Это упрощает получение данных для дальней работы с ними.
В форме Регистрации и Входа пользователя реализовано сокрытие пароля при вводе. Чтобы сделать вводимый пароль видимым, нужно поставить галочку у картинки с глазами.
Обработана кнопка крестик на всех панелях для выхода из приложения (кнопки не добавлялись, они прямо на формах), сообщение о подтверждении будет выходить 2 раза (на всякий случай).
Также имеется механизм для шифрования и дешифрования данных.
Файлы с данными о пользователя находится в user.json.
Внимание! 
Все папки пользователей при регистрации будут создататься корневой папке проекта в папке Авторы. Любые другие папки и файлы в них были созданы раньше до изменения пути записи, поэтому не обращаем на них внимание (для примера: Redaction\\Redaction\\bin\\Debug\\net6.0-windows\\Авторы\\anna" - правильное окончание пути для создания и сохранения)

Имеется файл Скрины работы программы, который содержит скрины работы приложения.
